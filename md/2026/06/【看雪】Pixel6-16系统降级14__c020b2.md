> 💡 **AI 总结（key-points）**
>
> 通过保持两个系统分区（A/B）的bootloader版本一致，可安全地将Pixel 6从Android 16降级至14，避免黑砖风险。
> 
> - **核心前提：** 必须确保手机A、B分区的bootloader版本相同，否则直接刷入旧版系统镜像会导致设备黑砖。
> - **版本检查与统一：** 通过`fastboot getvar version-bootloader`命令检查分区版本；若版本不一致，则需使用当前系统镜像中的bootloader文件，通过命令将两个分区都刷入统一版本。
> - **绕过版本校验：** 刷入目标Android 14系统镜像前，需编辑解压目录中的`android-info.txt`文件，删除其中的`version-bootloader`和`version-baseband`版本要求行。
> - **完成降级与后续操作：** 在修改文件后的目录中执行`fastboot flashall -w`完成刷机。成功后应进入开发者选项关闭系统自动更新，防止再次升级。

由于手机自动更新导致升级到16，bootloader也升级到了16，无法退回旧版本的bootloader查阅相关资料发现降级会导致黑砖，所以不降级bootloader只降级系统。

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782182816842-43152ea0-f663-4dc1-872c-ee80b3545f91.png)

#### 1.前提要保证a和b分区bootloader版本一致

版本不一致刷完有可能导致黑砖

```
# 重启进入bootloader模式
adb reboot bootloader

# 查看当前分区版本
fastboot getvar current-slot
fastboot getvar version-bootloader

#切换分区b （如果切换回去查看完，要记得切换回来）
fastboot --set-active=b
```

可以看到当前是a分区版本是 16.4-14548185

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181147501-7e7fd488-edc4-49ef-a775-69c881e82fc4.png)

切换到b分区，版本也是一样的（查看后，注意要切换回去a分区）

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181223926-ad64d374-bce7-4675-a72d-4046f8a5c3fe.png)

#### 2.如果bootloader版本不一样或者不确定，下载当前系统版本镜像，将a和b分区都刷一遍

```
# 将当前分区刷入bootloader
fastboot flash bootloader bootloader-oriole-slider-16.4-14548185.img
# 将另外一个分区刷入bootloader
fastboot --slot=other flash bootloader bootloader-oriole-slider-16.4-14548185.img
```

下载当前系统镜像包，找到bootloader镜像

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181273592-33a6b73b-b4ea-4b02-911b-f5188f43a673.png)

对2个分区都刷一遍

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181353681-42cde0e6-9fbc-40d1-b061-d165e4ce95de.png)

#### 3.刷入14系统镜像

将包解压

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181415059-9fcf2aa1-4781-405b-88dd-f73aefc95ac6.png)

打开解压后的文件夹，找到android-info.txt，将

require version-baseband=g5123b-120690-230802-B-10596884

require version-bootloader=slider-1.3-10674934

都删掉，再保存。

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181446684-b57b0e7b-7310-4df2-adb9-3f619277f8bd.png)

在解压的文件夹下执行刷机命令

```
fastboot flashall -w
```

会报错，设置系统环境到当前文件夹路径下即可。

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181555878-5347317b-f21a-4047-90d2-a5e65375ad57.png)

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782181613339-b935b91f-9f31-42c3-b755-fb4ab3abc233.png)

设置完成后，要重新cmd窗口打开，再执行一次fastboot flashall -w，刷完开机后，就降级成功了，再进入开发者模式把自动更新关了，以免下次自动更新。

![](⚠️ https://cdn.nlark.com/yuque/0/2026/png/46995430/1782183971857-18cb8143-f09c-4189-b8de-997105607b9c.png)
